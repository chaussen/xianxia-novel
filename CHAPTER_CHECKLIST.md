# 章节收尾 Checklist

> 每章定稿后，在 git commit 之前完成以下所有步骤。
> 全部打勾后再执行 `git commit -m "release: 第X章定稿"`

---

## 第一步：文件归档

- [ ] 草稿从 `drafts/` 移动到 `chapters/`，文件名格式：`0X_chapter.md`
- [ ] `drafts/` 中的物理方案书已删除或移至 `drafts/archive/`

---

## 第二步：更新 CHAPTER_LOG.md

- [ ] 在"已完成章节"下新增本章条目，包含：
  - 一句话摘要
  - 本章已用物理原理（地球名称 + 顾青命名）
  - 道具变化（消耗了什么、获得了什么、当前完整列表）
  - 人物状态变化（新出场人物、关系变化、死亡等）
  - 触发的世界事件

- [ ] 在"物理原理使用记录"表格中新增行

- [ ] 确认"未来可用原理"列表中已用的已标注 ✅

---

## 第三步：更新 WORLD_CALENDAR.md

- [ ] 本章发生的背景事件已更新状态（进行中→完结，或新增事件）
- [ ] "历史事件记录"表格已新增本章触发的事件
- [ ] "下一章可用场景素材"已更新，至少写2个可用素材

---

## 第四步：更新 MASTER_CODEX.md

- [ ] 第八节"当前状态追踪"已更新：
  - 已完成章节列表
  - 当前修为
  - 当前位置
  - 待解决的问题/装置
  - 已用物理原理
  - 当前世界事件

- [ ] 如本章引入新人物，已在第四节"人物档案"新增条目
- [ ] 如本章用了新物理原理，已在第五节"物理映射表"标注 ✅

---

## 第五步：Git 操作

```bash
# 确认所有改动
git status

# 暂存所有文件
git add .

# 提交（替换X为章节号）
git commit -m "release: 第X章定稿，更新世界日历和章节日志"

# 如果是在 feature branch 上，合并回主分支
git checkout main
git merge chapter-0X
git branch -d chapter-0X
```

---

## 快速自查（提交前30秒）

- [ ] 本章有没有用过之前已用的物理原理？（查 CHAPTER_LOG）
- [ ] 新出场角色有没有自己独立存在的理由？
- [ ] 顾青所有台词，古代人能理解吗？
- [ ] 逻辑员审查报告中的所有 ❌ 都已修复了吗？
- [ ] 状态监测（道具列表）和实际剧情一致吗？
